# Project Plans

- Store project-specific implementation plans as markdown files in `.claude/plans/`
- Plan files should be detailed enough for a Haiku model to implement without additional context
- Include exact code blocks, file paths, and step-by-step instructions
- Name files descriptively: `feature-name.md` (e.g., `trade-in-buyback-system.md`)
- Mark plan status in the frontmatter or first heading (e.g., "Ready for implementation", "In progress", "Completed")
- Update plans as implementation progresses — mark completed phases, note deviations
- Plans are project-scoped — do not store personal preferences or global patterns here
